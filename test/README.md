# 벽 자동 인식 회귀 가드

`wall-snapshots.json` = 픽스처 도면별 벽 인식 결과의 **좌표 단위 스냅샷**
(각 항목: "fromX,fromZ,toX,toZ,두께[,창N]" 정렬 목록).

## 프로토콜
- trace.js를 변경하면 픽스처 전체를 재실행해 이 스냅샷과 **좌표 단위로** 대조한다.
  개수 비교만으로는 재구성이 은폐된다(실사례: 밴드 룰 변경이 CAD 벽을 재구성했지만
  개수가 우연히 같아 가드를 통과했음).
- 차이가 나면: (1) 의도된 개선인지 판별하고 근거와 함께 스냅샷 갱신, (2) 아니면 롤백.
- 실전 지표는 별도: 합성 하드셋(ai-auto/도면샘플/실전풍_*) recall/precision + LH 배치 커버리지.

픽스처 원본: ai-auto/도면샘플/ (버전 관리 밖). widthCm은 스냅샷에 기록됨.

### 벤치 러너(재사용)
세션 스크립트 재작성 대신: 스크래치의 `bench_runner.mjs`(헤드리스 Chrome+CDP,
node 24 내장 WebSocket) + `score_all.py`(스냅샷→GT 변환·eval_segments 일괄 채점).
사용법: `public/bench/`에 픽스처 복사 → `npm run dev` → `node bench_runner.mjs
[v1|v2|v2old|both] [키...]`(BENCH_DEBUG=1이면 라벨맵·stats 저장) → `python3 score_all.py`.
원본은 도면샘플 폴더에 보관: `ai-auto/도면샘플/bench_runner.mjs`, `score_all.py`.
